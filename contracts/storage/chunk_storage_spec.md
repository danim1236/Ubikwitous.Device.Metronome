# Ubikwitous Chunk Storage Specification

This document defines the filesystem layout and filename conventions
used by Ubikwitous edge devices when storing video chunks.

The goal is to guarantee deterministic and scalable storage
for large deployments of cameras.

---

# 1. Storage Hierarchy

Chunks MUST be stored using the following directory hierarchy:

tenant_id /
    site_id /
        location_id /
              YYYY /
                  MM /
                      DD /

Example:

ten_1/
   sit_1/
      loc_23/
         2026/
            03/
               05/

---

# 2. Chunk Filename

Chunk filenames MUST represent the start timestamp of the chunk
in Unix epoch seconds.

Format:

chunk_{cam}_{timestamp-ms}_{skip}.h264


Example:

chunk_cam1_1710000030023_3.h264

This file represents a chunk starting at:

1710000030.023 seconds since Unix epoch

and have 3 frames before keyframe
---

3. Full Path Example

Example full path:

tenantA/mall1/store23/2026/03/05/chunk_cam1_1710000030023_3.h264


---

# 4. Chunk Duration

Chunks are generated using the configured duration:

chunk_duration_sec

Example:

chunk_duration_sec = 30


This means:

chunk_cam_{1}_1710000030023.h264 → chunk_cam_{1}_1710000060.023.h264

---

5. Chunk Ordering

Chunks MUST be strictly ordered by timestamp.

chunk_cam_{1}_1710000030023.h264
chunk_cam_{1}_1710000060026.h264
chunk_cam_{1}_1710000090026.h264


No gaps are expected unless:

- camera failure
- device reboot
- network interruption

---

# 6. Metadata Source

The metadata associated with each chunk is defined in:

contracts/protobuf/chunk.proto

The filesystem layout is a projection of the following fields:

tenant_id

site_id

location_id

device_id

camera_id

start_timestamp_ms

---

7. Clock Source

Chunk timestamps are generated by:

Ubikwitous.Device.Metronome


This ensures deterministic chunk boundaries across cameras.

---

# 8. Future Extensions

Future storage backends may include:

- S3
- object storage
- distributed filesystems

In such cases the path structure MUST remain identical.
